# Revature-Project-1
Build a full-stack application that satisfies the requirements set forth in the Expense Reimbursement Management System document.

## Overview
Project One is a full-stack **Expense Reimbursement Management System (ERMS)** built to satisfy the requirements defined in the project specification. The application supports employee expense submissions and managerial approval workflows.

The project is designed to demonstrate full-stack development skills, including RESTful API design, database integration, and a TypeScript-based front-end.

---

## Technology Stack

- **Backend:** Spring Boot (RESTful API)
- **Database:** PostgreSQL
- **Frontend:** React with TypeScript

---

## User Stories

### Employee Functionality
- Employees can register for a new account.
- Employees can log in to their account.
- Employees can submit reimbursement tickets with a price and description.
- Employees can view all past reimbursement tickets.
- Employees can view pending reimbursement tickets.
- Employees can view tickets ordered by date.

### Manager Functionality
- Managers can log in to their account.
- Managers can view all pending reimbursement requests.
- Managers can view past tickets for all employees within the company.
- Managers can accept or deny reimbursement requests.
